# 그리드 매매 (Grid Trading)

## 개념

그리드 매매는 **가격 구간을 일정 간격으로 나누고, 각 구간마다 매수/매도 주문을 미리 걸어두는 전략**이다.

가격이 내려가면 자동으로 사고, 올라가면 자동으로 파는 것을 반복한다.

## 작동 원리

### 예시: BTC 그리드 매매

- 가격 범위: $90,000 ~ $100,000
- 그리드 수: 5단계
- 투자금: $1,000

```
$100,000  ── 매도 주문 ──────────────  ↑ 여기서 팔고
 $97,500  ── 매도 주문 ──────────────  ↑ 여기서 팔고
 $95,000  ── 현재가 ─────────────────  ← 기준점
 $92,500  ── 매수 주문 ──────────────  ↓ 여기서 사고
 $90,000  ── 매수 주문 ──────────────  ↓ 여기서 산다
```

### 흐름

1. 현재가 기준으로 **위에는 매도 주문**, **아래에는 매수 주문**을 건다
2. 가격이 내려가서 $92,500 매수 주문이 체결되면 → 해당 위치에 매도 주문을 건다
3. 가격이 올라가서 $97,500 매도 주문이 체결되면 → 해당 위치에 매수 주문을 건다
4. 이 과정이 반복되면서 **각 구간마다 차익**을 얻는다

## 수익 구조

```
그리드 간격 = (상한가 - 하한가) / 그리드 수
그리드당 수익 = 그리드 간격 × 주문 수량
```

### 수익 예시

- 범위: $90,000 ~ $100,000 (간격 $10,000)
- 그리드 수: 10
- 그리드 간격: $1,000
- 1회 주문 금액: $100

→ 매수 후 매도 1회 = **약 $1.00 수익** (수수료 제외)
→ 하루에 10번 체결되면 = **약 $10/일**

## 장단점

### 장점

- 시장 방향을 예측하지 않아도 됨
- **횡보장(박스권)에서 가장 효과적**
- 감정 없이 자동으로 매매
- 24시간 작동

### 단점

- **하락장**: 계속 매수만 되어 물린다 (미실현 손실)
- **상승장**: 일찍 팔아버려서 큰 수익을 놓친다
- 범위를 벗어나면 매매가 멈춘다
- 수수료가 수익을 깎을 수 있다

## 핵심 설정값

| 설정 | 설명 | 팁 |
|------|------|----|
| 상한가 | 그리드 최고가 | 최근 저항선 참고 |
| 하한가 | 그리드 최저가 | 최근 지지선 참고 |
| 그리드 수 | 구간 분할 수 | 많을수록 체결 빈번, 수익은 작음 |
| 투자금 | 총 투자 금액 | 그리드 수 × 주문당 금액 |

## 리스크 관리

1. **손절 라인 설정**: 하한가 아래로 일정% 떨어지면 전체 포지션 정리
2. **투자금 제한**: 전체 자산의 일부만 사용
3. **범위 재설정**: 시장 상황 변하면 그리드 범위 조정

## Pacifica에서의 그리드 매매

- Pacifica는 **선물(Perpetual) 거래소**이므로 롱/숏 포지션으로 그리드 운영
- 레버리지 사용 가능하지만 **1x 권장** (초보자)
- ALO(Post Only) 주문을 사용하면 메이커 수수료 적용 (더 저렴)
- 배치 주문 API로 최대 10개 주문을 한번에 생성 가능

## 텔레그램 봇 명령어

```
/grid start <종목> <하한가> <상한가> <그리드수>   - 그리드 매매 시작
/grid stop                                       - 그리드 매매 중지
/grid status                                     - 현재 상태 확인
```

### 사용 예시

```
/grid start BTC 90000 100000 10
→ BTC $90,000~$100,000 구간, 10단계 그리드 매매 시작

/grid status
→ 활성 그리드: BTC | 범위: $90,000~$100,000 | 체결: 5회 | 수익: $4.50

/grid stop
→ 모든 주문 취소, 포지션 정리
```
